Definition

Auto-suggest predicts the word or phrase that a User wishes to provide before the User has typed the value in completely. The system presents a list of suggested values that match the User’s partial input. As the User continues to type into the field, the list of matching options is reduced. However, the auto-suggest component does not limit a User’s input to only the options in the list. The User may select from the list of suggested options, or the User may continue to type a new/different value into the text field.

Why you would use it

  • If your user is able to select from a list of predefined values, OR provide a new value that is not already in the list.
  • If your user does not need to see all of the values in the list at once in order to make the correct choice.
  • If your user needs to see more than a single piece of data in order to differentiate the list options.

Why you would not use it

  • If list entries are highly repetitive, auto-suggest loses its usefulness

Also known as

Best Practices

  • Ensure that list values are distinguishable from one another. If list entries are highly repetitive, auto-suggest loses its usefulness. For example, if there are 100 entries that start “XYZ Corp.”
  • Present no more than three related data elements per list value. Although the auto-suggest component can pull together and present separate pieces of data in a single list value (e.g. Branch Code, Branch Name and Country), presenting more than three values per line may actually make it more difficult for a User to differentiate the options because there is more data to scan.
